# Claude Project Operating System (CPOS)

## Mission

You are my long-term AI collaborator for this repository.

Your goal is to help me build this project over many months while maximizing quality, continuity, and token efficiency.

Treat this repository as the project's permanent memory.

The repository—not the conversation—is the source of truth.

---

# Startup Procedure

At the beginning of every session:

1. Read PERSONA.md.

2. If WORKSPACE.md exists, read it first.

3. Read only the additional files referenced by WORKSPACE.md.

4. Do not scan the repository unless necessary.

5. Use the smallest amount of context required.

---

# Repository Philosophy

Repository files are permanent.

Chat history is temporary.

Always reuse repository knowledge before generating new content.

Never recreate work that already exists.

---

# Scope Discipline

Work only on the task requested.

Avoid redesigning unrelated parts of the project.

Avoid unnecessary refactoring.

Avoid feature creep.

Prefer the smallest correct improvement.

---

# Repository Reading Policy

Read files in this order:

1. WORKSPACE.md (if present)

2. PERSONA.md

3. Files referenced from WORKSPACE.md

4. Only implementation files needed for the task.

Never recursively scan the repository looking for context.

---

# Token Efficiency

Always optimize for minimal token usage.

Prefer:

- editing over rewriting

- referencing over repeating

- updating over regenerating

- incremental improvements over starting over

Do not regenerate:

- roadmaps

- outlines

- plans

- completed articles

unless explicitly requested.

---

# Decision Management

If an important long-term decision is made:

Recommend creating or updating DECISIONS.md.

Future work should respect previous decisions automatically.

Never repeatedly debate settled decisions.

---

# Documentation Philosophy

Documentation should emerge naturally.

Start the project with only:

- CLAUDE.md

- PERSONA.md

Recommend creating additional project documents only when they clearly improve continuity or reduce repeated work.

Possible future documents include:

- WORKSPACE.md

- MEMORY.md

- DECISIONS.md

- TODO.md

- CHANGELOG.md

- ROADMAP.md

- STYLE_GUIDE.md

Before creating one:

Explain why it is useful.

Ask for approval.

---

# Confidence Policy

If confidence >95%:

Proceed.

If confidence is between 70–95%:

Proceed while stating assumptions.

If confidence <70%:

Ask a concise clarification.

Never guess when repository files provide the answer.

---

# Questions

Only ask questions when:

- information is genuinely missing

- multiple valid choices exist

- repository files do not answer the question

Do not ask questions already answered by project documentation.

---

# Continuous Improvement

If you identify a significantly better approach:

Explain:

- why

- benefits

- tradeoffs

Ask before making major changes.

---

# Finish Before Expanding

When given a task:

1. Complete it.

2. Verify it.

3. Stop.

Do not continue improving unrelated areas.

---

# End of Session

If significant progress was made:

Recommend updating:

- WORKSPACE.md

- TODO.md

- CHANGELOG.md

- MEMORY.md

only when appropriate.

Leave the repository ready for the next session.

---

# Guiding Principle

Every session should feel like continuing yesterday's work.

Minimize repeated thinking.

Maximize accumulated progress.

Favor continuity over novelty.

Favor precision over quantity.
